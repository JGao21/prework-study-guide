HTML

  • HyperText Markup Language handles the content, the structure, and the meaning of each element on the web page.
  • The Head element contains information about the webpage.
  • The Body element represents the visible content shown to the user.
  • The Title element defines the title for a webpage. It is extremely important for SEO.
  • The Style element deines the CSS styles associated with a website
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • !DOCTYPE html declares that the following document is in HTML.
  • html lang="en" indicates that the doc is written in English.
  • h1 is the opening tag for the body elemen.t
  • The p element represents a paragraph or a block text.
  • The ul, ol,and li, elements represent unordered lists, ordered list, and list itmes, which shows up as numbered lists.

CSS

  • Cascading Scale Sheets (CSS) is used to describe the presentation of the website that’s created with HTML.
  • There are three ways of styling a webpage using CSS, which is Inline CSS, and Internal and External CSS Style Sheet.
  • External CSS Style Sheet is the most common because it keeps the CSS on a seperate document.
  • To create a new CSS document in assets, first cd to assets, then use touch style.css.
  • * is a symbol to apply a rule to all elements visible on the page.
  • .card is a class selector that style each element in html.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch [name]: creates a new branch and switches to it.
  • git add -A: records the changes made to the file.
  • git commit -m “[description]”: describes the changes.
  • git pull origin main: pulls changes from locally stored branches and check if the code is up to date.
  • git push origin feature/starter-code: pushes the code to GitHub.
